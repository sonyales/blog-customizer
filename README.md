# Blog Customizer

**Blog Customizer** — React-приложение для настройки внешнего вида статьи. Через боковую панель пользователь может изменить шрифт, размер текста, цвет текста, цвет фона и ширину контента, а затем применить выбранные параметры или сбросить их к значениям по умолчанию.

Настройки не применяются мгновенно: пользователь сначала выбирает параметры в форме, а затем подтверждает изменения кнопкой «Применить». Это позволяет отделить состояние формы от состояния уже применённых стилей.

## Реализация

### Состояние настроек

В приложении используются два состояния:

* `draftState` — текущие значения, выбранные в форме;
* `appliedState` — значения, применённые к статье.

Пока пользователь меняет параметры в панели, внешний вид статьи не обновляется. При нажатии на кнопку «Применить» значения из `draftState` переносятся в `appliedState`. При сбросе оба состояния возвращаются к дефолтным настройкам.

### CSS-переменные

Стили статьи обновляются через CSS custom properties, которые передаются в корневой контейнер приложения:

```tsx
style={
  {
    '--font-family': appliedState.fontFamilyOption.value,
    '--font-size': appliedState.fontSizeOption.value,
    '--font-color': appliedState.fontColor.value,
    '--container-width': appliedState.contentWidth.value,
    '--bg-color': appliedState.backgroundColor.value,
  } as CSSProperties
}
```

Такой подход позволяет менять визуальные параметры страницы без добавления условной логики в компоненты.

### Форма параметров

Форма настроек построена на контролируемых компонентах. Каждый элемент формы обновляет только связанный с ним параметр, сохраняя остальные значения без изменений.

Для управления настройками используются переиспользуемые UI-компоненты: select, radio group, кнопки, разделители и текстовые элементы.

### Боковая панель

Панель настроек можно открыть и закрыть по кнопке. Также реализовано закрытие при клике вне панели: при открытом состоянии добавляется обработчик `mousedown`, который проверяет область клика и закрывает панель, если клик был выполнен снаружи.

## Запуск проекта

### Клонировать репозиторий

```bash
git clone https://github.com/sonyales/blog-customizer.git
cd blog-customizer
```

### Установить зависимости

```bash
npm install
```

### Запустить приложение

```bash
npm start
```

После запуска приложение откроется в браузере автоматически. По умолчанию dev server работает на порту `8080`.

## Сборка проекта

```bash
npm run build
```

Готовая сборка будет создана в директории `dist`.

## Storybook

Для просмотра UI-компонентов в изоляции используется Storybook.

```bash
npm run storybook
```

Storybook запускается на порту `6006`.

Для сборки Storybook:

```bash
npm run build-storybook
```
